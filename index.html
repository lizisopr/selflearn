Complete JavaScript Guide

Study the code snippets below. Each part explains how JS works in real projects.

1. Variables & Data Types

Goal: How to store information (Strings, Numbers, Booleans).

 // 'let' allows the value to change later
 let score = 0; 

 // 'const' is for values that never change
 const playerName = "Alex"; 

 // Data Types:
 let isGameOver = false; // Boolean (True/False)
 let items = ["Sword", "Shield"]; // Array (List of items)
 let playerStats = { hp: 100, mp: 50 }; // Object (Complex data)

Note: Use const by default, and let only if you know the value will change.

2. Conditionals (If/Else)

Goal: Making decisions in code.

 if (score >= 100) {
    console.log("Level Up!"); // Runs if condition is true
 } else {
    console.log("Keep playing..."); // Runs if condition is false
 }

Note: Conditionals are the "brain" of your app—they decide what happens next.

3. Loops (Repeat Actions)

Goal: To run the same code many times without typing it again.

 // Start at 0; Run while i < 5; Add 1 each time
 for (let i = 0; i < 5; i++) {
    console.log("Current number is: " + i);
}

Note: Loops save time when dealing with lists (Arrays) or repeating tasks.

4. Functions (Reusable Code)

Goal: Create a "package" of code that you can call anytime.

 // Classic Function
 function multiply(a, b) {
    return a * b;
 }

  // Modern Arrow Function (Shorter syntax)
  const add = (a, b) => a + b;

  console.log(add(5, 10)); // Output: 15

Note: Arrow functions are the modern way to write functions in JS (ES6+).

5. DOM & Web APIs

Goal: Connect JS to HTML and the Internet.

 // DOM: Changing HTML content
 document.getElementById("title").innerText = "New Title!";

 // Web API: Getting data from a server (Fetch)
 fetch("https://api.example.com/data")
    .then(response => response.json())
    .then(data => console.log(data));

Note: The DOM makes websites interactive. APIs allow your site to talk to other services.
